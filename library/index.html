2. DeepCortex (DS) Assets¶

DeepCortex allows users to create and manage different data science Asset types. The Library Module is the primary asset management module, where users can import, edit, delete these different assets within DeepCortex. Library Module’s User Interface consists of 3 main panels:

  1. top-left panel: users can click the various action buttons here to create individual assets in DeepCortex (e.g. Create Album, Create Pipeline, Create DC Project, Create Experiment).

  2. bottom-left panel: users can navigate to differnt Asset Category here (Albums, Tables, Models, Pipelines), which are further categorized using the following taxanomy:

    1. All: all the assets within the selected Asset Category

    2. Personal: all the assets that have been imported, created by the user

    3. Shared w/Me: all the assets that have been shared with the user by other users of DeepCortex

    4. Enterprise: all the assets that have been shared with the user by the Enterprise (subject to Enterprise’s access control)

    5. Third Party: all third-party assets that have been made available to the users of DeepCortex

    6. Recent: recently viewed assets

  3. centre panel: users can view the list of different assets that are within the selecte Asset Category. On the top of the centre panel, are various action icons, that allows users to take specific actions on the individual assets (e.g Delete, Share, Clone etc.)

2.1 What are different Asset Types in DeepCortex?¶

Data Assets¶

Data Assets in DeepCortex are stored as 1 of 3 asset types – Albums, Tables, and Binary Datasets.

Album asset type is used to store images (png, npy, jpeg) and corresponding image annotations (in VOC format), as well as Full Motion Videos (FMVs).

Table asset type is used to store tabular data (csv)

Binary Datasets asset type is used to store data as BLOB.

Model Assets¶

Models created within or uploaded to DeepCortex are stored in the asset types Models, CV Models. To learn more about Models in DeepCortex see 6. Working with ML Models

Experimentation Assets¶

Operators are logic blocks that perform independent operation within DeepCortex. Operators are developed and published within DeepCortex so that they can be used in pipelines. To learn more about Operators in DeepCortex see 5. Designing Your Own Operators

Pipelines are assets constructed by stitching various Operators into a Directional Acyclic Graph (DAG). Pipelines in DeepCortex can be constructed to perform various tasks such as: Data Processing, AI Training, AI prediction etc. To learn more about Operators in DeepCortex see 3. Pipelines

2.3 How to share Assets in DeepCortex?¶

Users of DeepCortex can share assets with each other. Sharing assets is as simple as selecting the asset in Library and clicking the “Share” icon on the top of the centre panel. User will then be prompted to provide the recipient user’s email address (email address should be the same as what was used during account setup for the receipient user) with whom the specific asset is being shared.
